# WM 2026 — FIFA World Cup 2026 Result Predictor

A machine-learning pipeline (in R) that predicts the outcome of FIFA World Cup
2026 matches and Monte-Carlo simulates the entire tournament. It learns from
~49,000 historical international matches, rates teams with an Elo system plus
recent-form features, and trains a gradient-boosted classifier to estimate
**P(home win) / P(draw) / P(away win)** for every fixture. Those probabilities
then drive a full bracket simulation (group stage → final) to produce
tournament-wide advancement and title odds for all 48 teams.

---

## Results at a glance

Held-out evaluation on international matches from **2021 onwards** (the model
never sees them during training):

| Model                     | Accuracy | Log-loss | Brier |
|---------------------------|:--------:|:--------:|:-----:|
| **xgboost (this model)**  | **0.61** | **0.87** | **0.51** |
| Baseline: class priors    |   0.48   |   1.05   | 0.63  |
| Baseline: majority class  |   0.48   |  17.97   | 1.04  |

The model beats both naive baselines on every metric — ~61% three-way accuracy
is in the realistic range for international football (draws are inherently hard).
`elo_diff` is by far the most important feature, as expected.

official WC-2026 bracket (group stage → R32 → R16 → QF → SF → final), the
title favourites come out as:

| Argentina  | 28.0 %  |   38.1 %   | 50.6 %  |  80.9 %  |
| Spain      | 13.5 %  |   27.3 %   | 39.6 %  |  70.4 %  |
| Brazil     | 10.7 %  |   18.4 %   | 35.3 %  |  73.9 %  |
| Colombia   |  9.0 %  |   15.8 %   | 23.9 %  |  65.8 %  |
| France     |  8.7 %  |   19.0 %   | 36.5 %  |  80.4 %  |
| England    |  7.0 %  |   12.5 %   | 29.2 %  |  73.6 %  |
| Portugal   |  5.3 %  |   11.8 %   | 20.2 %  |  67.4 %  |

Full results for all 48 teams are written to
`output/tournament_probabilities.csv`. As a sanity check the simulation is
internally consistent: per round the probabilities sum to exactly the number of
slots (Win = 100 %, finalists = 200 %, …, Round of 32 = 3200 %).

> ⚠️ The model is **Elo-driven**, so it concentrates probability on the
> strongest teams — title odds for the top favourites run a little higher than
> the betting market. Treat these as model estimates, not forecasts.

> ℹ️ All predictions are written to `output/` (`fixture_predictions.csv`,
> `group_stage_simulation.csv`, `tournament_probabilities.csv`). They are
> regenerated by the pipeline and git-ignored.

## How to run

Requires **R ≥ 4.1**. From the project root:

```bash
make setup     # install required packages (first run only)
make all       # data -> train & evaluate -> predict -> simulate tournament
```

…or without `make`:

```bash
Rscript scripts/00_setup.R
Rscript run_all.R
```

…or one stage at a time:

```bash
Rscript scripts/01_build_dataset.R      # download + feature engineering
Rscript scripts/02_train_evaluate.R     # time-split evaluation + final model
Rscript scripts/03_predict_tournament.R # fixture predictions + group sim
Rscript scripts/04_simulate.R           # full-tournament Monte-Carlo (N=10,000)
Rscript scripts/04_simulate.R 100       # ...or override N for a quick run
```

The pipeline is split so each file has a single responsibility — data readers,
features, model, and prediction are independent and individually testable.

## Modelling details

- **Target:** 3-class match result from the home team's perspective
  (`home_win` / `draw` / `away_win`).
- **Features** (all computed *before* each match to avoid leakage):
  - `elo_*` — pre-match Elo ratings and their difference. Elo uses a
    goal-difference multiplier and a home-advantage term (0 for neutral
    venues, including all World Cup matches).
  - `form_*_diff` — difference in rolling points / goals-for / goals-against
    over each team's last `FORM_WINDOW` (= 5) matches.
  - `rest_diff` — difference in days since each team's previous match.
- **Recency weighting:** training rows are weighted by
  `exp(-decay · days_before_latest)` (~3.8-year half-life), so recent form
  counts more than results from a decade ago.
- **Algorithm:** `xgboost` multiclass (`multi:softprob`). Missing form values
  (a team's first few matches) are handled natively by xgboost.
- **Evaluation:** strictly time-based split (train on the past, test on the
  future) — never random CV, which would leak future information into a time
  series. Scored with accuracy, multiclass log-loss and Brier score against
  majority-class and class-prior baselines.
- **Group simulation (stage 03):** a quick stand-alone Monte-Carlo of just the
  group stage (`SIM_N` = 2000) reporting each team's probability of winning its
  group / advancing, plus expected points.
- **Full tournament simulation (stage 04, [`R/monte_carlo.R`](R/monte_carlo.R)):**
  the whole bracket is simulated `TOURNAMENT_SIM_N` (= 10,000) times:
  - **Group stage** is sampled and vectorised across all N runs at once. Points
    are 3/1/0; the tie-breaker is **higher Elo** (encoded as `points + Elo
    fraction`, since a W/D/L model has no goal difference — see *Limitations*).
  - **Qualification** follows the WC-2026 format: top 2 of each of the 12 groups
    (24 teams) + the 8 best third-placed teams.
  - **Bracket routing** is read directly from the fixtures API's slot labels
    (`Winner Group E`, `3rd Group A/B/C/D/F`, `Winner Match 74`, …) — i.e. the
    *official* FIFA 2026 bracket, not a hand-coded guess. Third-placed teams are
    routed to their eligible slots via bipartite matching (Kuhn's algorithm).
  - **Knockout matches** can't draw, so the draw probability is redistributed
    onto the two sides in proportion to their win odds
    (`P_A + P_draw · P_A/(P_A+P_B)`) before sampling the winner.
  - Aggregated over N runs into per-team probabilities of winning the group,
    reaching each round, and lifting the trophy.

All knobs live in [`R/config.R`](R/config.R).

## Limitations & possible next steps

- **No goal difference.** The model predicts W/D/L, not scorelines, so group
  ties are broken by Elo rather than by FIFA's real goal-difference / goals-for
  rules. A **Poisson goals model** (predict scorelines, then derive results and
  proper tie-breakers) is the most valuable next step.
- **Third-place routing** uses a *valid* bipartite matching consistent with each
  slot's group-eligibility, not FIFA's exact published permutation table. This
  only affects which specific slot a third-placed team lands in, not which teams
  advance — a negligible effect on aggregate probabilities.
- **Elo-driven concentration.** Because strength enters mainly through Elo, the
  simulation amplifies favourites across seven rounds; top-team title odds sit a
  little above the betting market.
- **All group matches are re-sampled**, including the 8 already played; the
  finished results feed the Elo ratings but are not pinned as fixed outcomes.
  Conditioning the sim on known results is an easy refinement.
- No player-level information (injuries, squad strength, line-ups); home
  advantage is 0 for all WC matches (host nations treated as neutral).
- A reproducible environment lock (`renv`) would round out the project.
