# WM 2026 — FIFA World Cup 2026 Result Predictor

A machine-learning pipeline (in R) that predicts the outcome of FIFA World Cup
2026 matches and simulates the group stage. It learns from ~49,000 historical
international matches, rates teams with an Elo system plus recent-form features,
and trains a gradient-boosted classifier to estimate
**P(home win) / P(draw) / P(away win)** for every fixture.

---

## Results at a glance

Held-out evaluation on international matches from **2021 onwards** (the model
never sees them during training):

| Model                     | Accuracy | Log-loss | Brier |
|---------------------------|:--------:|:--------:|:-----:|
| **xgboost (this model)**  | **0.61** | **0.87** | **0.51** |
| Baseline: class priors    |   0.48   |   1.05   | 0.63  |
| Baseline: majority class  |   0.48   |  17.97   | 1.04  |

The model beats both naive baselines on every metric — ~61% three-way accuracy
is in the realistic range for international football (draws are inherently hard).
`elo_diff` is by far the most important feature, as expected.

> ℹ️ Predictions are written to `output/` (e.g. `fixture_predictions.csv`,
> `group_stage_simulation.csv`). These are regenerated by the pipeline and are
> git-ignored.

## How to run

Requires **R ≥ 4.1**. From the project root:

```bash
make setup     # install required packages (first run only)
make all       # build dataset -> train & evaluate -> predict
```

…or without `make`:

```bash
Rscript scripts/00_setup.R
Rscript run_all.R
```

…or one stage at a time:

```bash
Rscript scripts/01_build_dataset.R     # download + feature engineering
Rscript scripts/02_train_evaluate.R    # time-split evaluation + final model
Rscript scripts/03_predict_tournament.R# fixture predictions + group sim
```

## Project structure

The pipeline is split so each file has a single responsibility — data readers,
features, model, and prediction are independent and individually testable.

```
WM_2026/
├── R/                       # library code (functions only, no side effects)
│   ├── config.R             # all paths, URLs, hyper-parameters
│   ├── utils.R              # small shared helpers
│   ├── team_mapping.R       # reconcile team names across sources
│   ├── data_reader.R        # READ: WC-2026 fixtures API  -> tidy matches
│   ├── historical_data.R    # READ: historical results CSV -> tidy matches
│   ├── features.R           # Elo ratings + rolling-form feature engineering
│   ├── model.R              # MODEL: train / predict / persist (xgboost)
│   ├── evaluate.R           # time-split metrics vs. baselines
│   └── predict.R            # fixture predictions + Monte-Carlo group sim
├── scripts/                 # drivers that compose the library into a pipeline
│   ├── 00_setup.R           #   install dependencies
│   ├── 01_build_dataset.R   #   readers + features  -> training_data.rds
│   ├── 02_train_evaluate.R  #   evaluate + fit final -> result_model.rds

├── run_all.R                # runs stages 01 -> 02 -> 03
├── Makefile                 # make setup | data | train | predict | all
├── data/                    # raw cache + processed datasets (git-ignored)
└── output/                  # predictions, metrics, saved model (git-ignored)
```

## Modelling details

- **Target:** 3-class match result from the home team's perspective
  (`home_win` / `draw` / `away_win`).
- **Features** (all computed *before* each match to avoid leakage):
  - `elo_*` — pre-match Elo ratings and their difference. Elo uses a
    goal-difference multiplier and a home-advantage term (0 for neutral
    venues, including all World Cup matches).
  - `form_*_diff` — difference in rolling points / goals-for / goals-against
    over each team's last `FORM_WINDOW` (= 5) matches.
  - `rest_diff` — difference in days since each team's previous match.
- **Recency weighting:** training rows are weighted by
  `exp(-decay · days_before_latest)` (~3.8-year half-life), so recent form
  counts more than results from a decade ago.
- **Algorithm:** `xgboost` multiclass (`multi:softprob`). Missing form values
  (a team's first few matches) are handled natively by xgboost.
- **Evaluation:** strictly time-based split (train on the past, test on the
  future) — never random CV, which would leak future information into a time
  series. Scored with accuracy, multiclass log-loss and Brier score against
  majority-class and class-prior baselines.
- **Tournament simulation:** each group match is sampled from its predicted
  probabilities `SIM_N` (= 2000) times; qualification follows the WC-2026 rule
  (top 2 of each group + the 8 best third-placed teams). Output is each team's
  probability of winning its group / advancing, plus expected points.

All knobs live in [`R/config.R`](R/config.R).

## Limitations & possible next steps

- The knockout bracket is **not** simulated end-to-end: the fixtures API uses
  placeholder slots for knockout teams, so only the group stage is simulated.
  A full bracket Monte-Carlo (re-seeding R32 → final from group outcomes) is the
  natural next extension.
- No player-level information (injuries, squad strength, line-ups) — the model
  is purely team-level.
- Home advantage is set to 0 for all World Cup matches; host nations
  (USA / Mexico / Canada) playing at home are treated as neutral.
- A Poisson goals model (predicting scorelines, then deriving results) and a
  reproducible environment lock (`renv`) would both be sensible additions.
